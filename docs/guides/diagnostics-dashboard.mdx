The diagnostics dashboard lives alongside the [web control panel](https://docs.almond.bot/guides/control-panel) at `/diagnostics` — open [axol.almond.bot/diagnostics](https://axol.almond.bot/diagnostics) (or `https://<axol-host>:8001/diagnostics` on the LAN) while `axol serve` is running.

## Motor status

A card per motor (8 per arm) shows its status code, temperature, bus voltage, and live position, refreshed by the server's 1 Hz health ping. Clicking a card opens the full `motor.info` readout — model, firmware, control mode, gains, and live readings — without launching a CLI session. The **CAN setup** and **CAN enable** buttons in this section run the bring-up commands ([`can.setup`](https://docs.almond.bot/cli/can-setup) / [`can.enable`](https://docs.almond.bot/cli/can-enable)) ad hoc when motors are unreachable.

## Live telemetry

Position, velocity, and torque charts for the selected arm stream continuously at 10 Hz while the robot link is connected. The server keeps a 10-minute rolling buffer, so the charts backfill on page load. The filter row scopes all three charts: time window presets (30s–10m), arm, and per-joint toggles; scroll to zoom and drag to pan, which pins the view until you press **Go live**.

Telemetry is sampled by the idle robot link, which is the single owner of the CAN bus. While a diagnostic or operation owns the bus the stream pauses and the charts label the gap — history is kept, and streaming resumes as soon as the bus is released.

## Diagnostics

Each diagnostic — [`diag.home`](https://docs.almond.bot/cli/diag-home), [`diag.gripper`](https://docs.almond.bot/cli/diag-gripper), [`diag.rom-enable`](https://docs.almond.bot/cli/diag-rom-enable), [`diag.rom-disable`](https://docs.almond.bot/cli/diag-rom-disable) — is a card: click it, set parameters in the dialog, and hit Run. The card shows the running state and elapsed time; completion lands as a notification, and the full log is kept with the run in the history below. Interactive prompts are replaced by countdowns automatically (`--no-prompt`), and launching hands the CAN bus over to the diagnostic for the duration of the run. (The `motor.info` / `motor.health` read set needs no action here — the motor cards above show it live.)

## Run history

Every diagnostic launched from the dashboard is recorded as a *run*: command, arguments, duration, exit status, log, and the telemetry observed while it executed (persisted under `~/.almond/diagnostics/runs` on the serve host). The ROM soak test additionally writes its own 5 Hz position/torque capture, since it owns the CAN bus while it runs — opening a run charts its capture with the same joint filter and zoom/pan as the live charts.
