## Requirements

- **Linux**
- **Python 3.13+** (the quick install below provides it automatically)
- **(Optional) NVIDIA Jetson** (e.g. a ZED Box) — required for the GMSL-attached ZED cameras (data collection / policy inference).

## Quick install (recommended)

One command installs everything and starts the web control panel:

```bash
curl https://axol.almond.bot/install -fsS | bash
```

This:

- Installs [`uv`](https://docs.astral.sh/uv/) if it isn't already installed.
- Installs the `axol` CLI from GitHub as a `uv` tool (Python 3.13 included), with every dependency extra except `cuda`.
- Sets up the headset-video stack ([`gst.install`](/cli/gst-install) plus the patched ZED source plugins, [`gst.build-zed`](/cli/gst-build-zed)) and, on a Jetson, pins the real-time clocks ([`jetson.setup`](/cli/jetson-setup)).
- Registers a root systemd service (`axol.service`) that runs [`axol serve`](/cli/serve) at startup (re-pinning the Jetson clocks first) and restarts it automatically if it ever goes down.

When it finishes, open [axol.almond.bot](https://axol.almond.bot) and connect to the machine — no terminal needed from then on. The install keeps itself in sync with the latest release: whenever the control panel connects, the server checks for updates in the background and restarts onto the new version once idle. (Re-running the install command also upgrades.)

```bash
journalctl -u axol -f        # service logs
systemctl restart axol       # manual restart
```

<Note>
  The service runs as root, so privileged steps (CAN bring-up, ZED daemon restarts) need no sudo password.
</Note>

## Development install

For working on Axol itself, install from a clone using [`uv`](https://docs.astral.sh/uv/). `pyroki` and `lerobot` are sourced from Git and resolved automatically:

```bash
uv sync
```

Then activate the virtual environment so the `axol` CLI is on your path:

```bash
source .venv/bin/activate
```

<Tip>
  You need to activate the environment in every new shell session. Alternatively, prefix every command with `uv run` (e.g. `uv run axol can.setup`) to skip activation.
</Tip>

### Optional extras

Install optional dependency groups as needed (the quick install above includes all of them except `cuda`):

| Extra | Contents | When to use |
|---|---|---|
| `lerobot` | LeRobot (from GitHub) | `collect-data`, `run-policy` |
| `sim` | viser | `teleop --sim` |
| `cuda` | JAX with CUDA 13 support | GPU-accelerated JAX (IK solver used by `teleop`); note that CPU is usually faster for the JAX IK solver |

```bash
uv sync --extra lerobot --extra sim          # teleoperation + data collection
uv sync --extra lerobot --extra cuda         # policy execution on GPU
uv sync --extra lerobot --extra sim --extra cuda   # everything
```

<Note>
  Streaming the ZED camera feeds to the headset (`teleop --cameras`, `collect-data`) isn't a dependency extra — it needs the system GStreamer NVENC tools. Install them with [`axol gst.install`](/cli/gst-install); see [Headset video](#headset-video-nvenc) below.
</Note>

## ZED Python bindings

The ZED Python bindings (`pyzed`) are not on PyPI and must be installed separately after the ZED SDK is installed:

```bash
axol zed.install
```

See [`zed.install`](/cli/zed-install) for details.

## Headset video (NVENC)

Relaying the ZED cameras to the headset (`teleop --cameras`, `collect-data`) grabs and **NVENC**-encodes frames entirely on the GPU via the zed-gstreamer pipeline (`zedxonesrc` / `zedsrc`), and sends them over WebRTC with **aiortc** (a normal Python dependency). This path needs the system GStreamer + GObject-introspection packages and **PyGObject** (built into the axol environment, like `pyzed`), which aren't a PyPI extra. Install them once:

```bash
axol gst.install
```

The ZED source elements (`zedxonesrc` / `zedsrc`) aren't part of that stack — build the patched versions (sensor-accurate frame timestamps) once, with the ZED SDK present:

```bash
axol gst.build-zed
```

On a Jetson, also pin the NVENC/VIC clocks for low-latency encode (the quick install does this at boot via systemd):

```bash
axol jetson.setup
```

The quick install runs all three for you. See [`gst.install`](/cli/gst-install), [`gst.build-zed`](/cli/gst-build-zed), and [`jetson.setup`](/cli/jetson-setup).

## Web control panel (optional)

With the quick install, the control panel is already running — just open [axol.almond.bot](https://axol.almond.bot) and connect to the machine.

For a development install, you can build and serve the UI locally instead. This produces the bundle that [`axol serve`](/cli/serve) hosts at `web/app/dist`:

```bash
cd web
npm install
npm run build --workspace=packages/axol-vr-client   # client package first
npm run build --workspace=app                        # → web/app/dist
```

Then start the server and open the printed URL:

```bash
axol serve
```

The same build also serves the [VR interface](/guides/vr-interface). `axol serve` works without this step (the API is still available) — the hosted UI at [axol.almond.bot](https://axol.almond.bot) connects to it directly. See the [Web Control Panel guide](/guides/control-panel).

## First-run hardware setup

Before using any motor or robot commands, initialize the CAN hardware:

```bash
axol can.setup
```

<Note>
  Run directly in a terminal, `sudo` is invoked automatically where required (with the quick install, the control panel runs this for you as root on first connect). See [`can.setup`](/cli/can-setup) for what it configures.
</Note>
